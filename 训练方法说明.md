This document describes a **single-step, advantage-weighted training framework** for jointly training
a hierarchical classification agent and a summary update agent under **delayed, trajectory-level rewards**.
The method avoids trajectory-level preference learning and instead adopts a stable, step-wise learning strategy.

---

## 2. Problem Formulation

### 2.1 State Definition

We consider two agents: a **classification agent** and a **summary update agent**.
In both cases, the *state* corresponds exactly to the textual prompt fed into the model.

#### Classification Agent State

For a document \(x\), at a node \(n\) in the taxonomy tree:

s = (
    topic,
    summary(n),
    summaries(children(n)),
    document x
)


#### Summary Update Agent State

When updating the summary of node \(n\):

s = (
    topic,
    summary(n),
    summary(parent(n)),
    summaries(siblings(n)),
    changed_content
)


In both cases, the state \(s\) is fully represented as text and is directly consumable by a language model.

---

### 2.2 Action Definition

#### Classification Agent Action

The classification agent outputs a **joint multi-binary decision**:

a = [
    class_1: Yes/No,
    class_2: Yes/No,
    ...
    class_k: Yes/No,
    NEW: Yes/No
]


Each output line corresponds to a binary classification decision for an existing child category or the creation
of a new category (`NEW`).

#### Summary Update Agent Action

The summary agent outputs:

a = (update_flag, updated_summary?)


- `update_flag ∈ {Yes, No}`
- `updated_summary` is generated only if `update_flag = Yes`

---

### 2.3 Trajectory Definition

Processing a document produces a **trajectory**:

\[
\tau = \{(s_1, a_1), (s_2, a_2), \dots, (s_T, a_T)\}
\]

A trajectory consists of:
- a **top-down classification phase**, followed by
- a **bottom-up summary update phase**.

Crucially, the quality of individual actions cannot be evaluated independently; feedback is only available
after the full trajectory is completed.

---

## 3. Role of the Global Reward \(R_{\text{global}}\)

Each trajectory \(\tau\) is assigned a **single scalar global reward**:

\[
R_{\text{global}}(\tau) = R_{\text{margin}}(\tau) + R_{\text{len}}(\tau)
\]

where:
- \(R_{\text{margin}}\) measures classification confidence margins against ground-truth-aligned labels,
- \(R_{\text{len}}\) penalizes unnecessarily long or complex trajectories.

Important properties of \(R_{\text{global}}\):

- **Delayed**: computed only after the trajectory terminates.
- **Non-differentiable**: relies on external judges or frozen models.
- **Trajectory-level**: does not directly assess individual actions.

As such, \(R_{\text{global}}\) cannot be used as a supervised label for step-wise training.

---

## 4. From Global Reward to Step-level Learning Signal

### 4.1 Motivation

Within a trajectory:
- some actions are critical,
- some are redundant,
- some may even be locally suboptimal but globally recoverable.

Therefore, directly treating all actions in a high-reward trajectory as "correct"
(or all actions in a low-reward trajectory as "incorrect") introduces excessive noise.

To address this, we use the global reward only as a **delayed evaluation signal**
to compute *advantages* for step-level actions.

---

### 4.2 Advantage Definition

For a state-action pair \((s, a)\) that appears in trajectory \(\tau\),
we define its advantage as:

\[
A(s, a) = R_{\text{global}}(\tau) - b(s)
\]

where \(b(s)\) is a **state-dependent baseline**, representing the expected
global reward when acting from state \(s\).

---

### 4.3 Baseline Estimation

In practice, the baseline is computed empirically as:

\[
b(s) = \mathbb{E}_{\tau \sim \mathcal{D}}[R_{\text{global}}(\tau) \mid \tau \text{ contains } s]
\]

That is, for each state \(s\), we average the global rewards of all trajectories
that pass through \(s\).

This is made possible by sampling **multiple action branches from the same state**.

---

## 5. Data Construction Procedure

### Step 1: Trajectory Sampling

For each document:
- Perform a rollout using the current classification and summary agents.
- At each state, sample multiple candidate actions.
- Branch trajectories at the current step only (no recursive branching).

This yields a **trajectory tree**, whose leaves correspond to complete trajectories.

---

### Step 2: Global Reward Computation

For each completed trajectory \(\tau\):
- Construct the final taxonomy tree.
- Compute \(R_{\text{global}}(\tau)\) using the predefined reward function.

---

### Step 3: Step-level Dataset Construction

Initialize an empty dataset \(\mathcal{D}_{\text{step}}\).

For each trajectory \(\tau\):

for (s_t, a_t) in τ:
add (s_t, a_t, R_global(τ)) to D_step


At this stage, each step-level sample is annotated with the global reward
of the trajectory it belongs to.

---

### Step 4: Advantage Computation

Group samples in \(\mathcal{D}_{\text{step}}\) by state \(s\).

For each group:
1. Compute baseline \(b(s)\) as the mean of associated global rewards.
2. Compute advantage:
   \[
   A(s, a) = R_{\text{global}} - b(s)
   \]

---

### Step 5: Filtering and Weighting

To improve stability:
- Discard samples with \(|A(s,a)| < \varepsilon\),
- Optionally keep only samples with \(A(s,a) > 0\).

Define the training weight as:

\[
w(s,a) = \exp\left(\beta \cdot \text{clip}(A(s,a), A_{\min}, A_{\max})\right)
\]

where \(\beta\) controls the sharpness of reweighting.

The final training dataset consists of weighted step-level samples:

\[
\mathcal{D}_{\text{train}} = \{(s, a, w)\}
\]

---

This dataset can be directly used for **advantage-weighted supervised fine-tuning**
of both the classification and summary agents.
