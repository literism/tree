# 引用丰富程序 - 性能优化说明

## 性能优化

### 并发处理

程序现在使用**异步并发**处理引用，大幅提升速度：

#### 优化前（串行处理）
```
引用1 → 引用2 → 引用3 → 引用4 → ...
每个引用需要 2-5 秒
100个引用 ≈ 200-500 秒
```

#### 优化后（并发处理）
```
引用1 ↘
引用2 → 并发处理 → 更快完成
引用3 ↗
...
100个引用 ≈ 20-50 秒（10个并发）
```

### 配置参数

在 `enrich_references.py` 中可调整的性能参数：

```python
MAX_CONCURRENT_REQUESTS = 10  # 最大并发请求数（可调整为5-20）
REQUEST_TIMEOUT = 15           # 单个请求超时时间（秒）
DELAY_BETWEEN_REQUESTS = 0.0   # 请求间延迟（0表示无延迟）
MIN_CONTENT_LENGTH = 300       # 最小内容长度（字符）
```

### 性能调优建议

#### 1. 调整并发数

**保守模式**（稳定但慢）：
```python
MAX_CONCURRENT_REQUESTS = 5
```

**标准模式**（推荐）：
```python
MAX_CONCURRENT_REQUESTS = 10
```

**激进模式**（快但可能被限制）：
```python
MAX_CONCURRENT_REQUESTS = 20
```

#### 2. 调整超时时间

如果网络较慢：
```python
REQUEST_TIMEOUT = 30  # 增加到30秒
```

如果网络很快：
```python
REQUEST_TIMEOUT = 10  # 减少到10秒
```

#### 3. 添加请求延迟

如果遇到被限制/封禁：
```python
DELAY_BETWEEN_REQUESTS = 0.5  # 每个请求延迟0.5秒
```

### 预估处理时间

假设：
- 每个topic平均 100 个引用
- 每个引用处理时间 2-5 秒
- 并发数 10

**计算**：
```
单个topic: 100引用 × 3秒 / 10并发 = 30秒
1000个topics: 30秒 × 1000 = 30000秒 ≈ 8.3小时
```

### 实时监控

程序运行时会显示：
```
[1/100] Albert Einstein (340 个引用)
  进度: 127/340 | ✓ 成功: 98 | ✗ 失败: 29
```

- **进度**：当前完成的引用数
- **✓ 成功**：成功爬取内容的引用数
- **✗ 失败**：失败的引用数（包括各种失败原因）

### 断点续传

程序支持断点续传：
- 每完成一个topic，立即保存进度
- 如果中断，重新运行会自动跳过已处理的topics
- 可以随时中断（Ctrl+C）并稍后继续

### 优化技巧

1. **分批处理**：如果数据量很大，可以先处理一部分topics进行测试
2. **监控资源**：注意CPU和内存使用，如果过高可以降低并发数
3. **网络稳定性**：使用稳定的代理可以提高成功率
4. **错误率监控**：如果失败率过高，考虑降低并发数或增加延迟

### 故障排查

#### 问题：速度仍然很慢
- 检查网络连接速度
- 检查代理是否正常工作
- 尝试增加 `MAX_CONCURRENT_REQUESTS`

#### 问题：大量请求失败
- 降低 `MAX_CONCURRENT_REQUESTS` 到 5
- 增加 `DELAY_BETWEEN_REQUESTS` 到 0.5-1.0
- 检查代理是否被限制

#### 问题：程序崩溃或卡住
- 降低 `MAX_CONCURRENT_REQUESTS` 到 5
- 减少 `REQUEST_TIMEOUT` 到 10
- 检查系统资源（内存、CPU）

### 性能对比

| 配置 | 并发数 | 延迟 | 100引用用时 | 适用场景 |
|------|--------|------|-------------|----------|
| 保守 | 5 | 0.5s | ~60秒 | 网络不稳定，容易被限制 |
| 标准 | 10 | 0s | ~30秒 | 一般情况（推荐） |
| 激进 | 20 | 0s | ~15秒 | 网络稳定，不担心限制 |

### 注意事项

1. **合理使用**：过高的并发可能导致IP被封禁
2. **尊重目标网站**：避免对同一网站发起过多请求
3. **资源消耗**：高并发会占用更多内存和CPU
4. **成功率权衡**：速度越快，失败率可能越高
